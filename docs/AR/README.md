# الهندسة الاحترافية بمساعدة الذكاء الاصطناعي باستخدام .NET — نظام التصدير

## نظرة عامة

نظام التصدير (Export Pipeline) هو نظام احترافي لإنشاء مخطوطات الكتاب
*الهندسة الاحترافية بمساعدة الذكاء الاصطناعي باستخدام .NET*. يقوم بتحويل
ملفات Markdown المصدرية إلى مستندات PDF و DOCX جاهزة للنشر باستخدام Pandoc
كمحرك تحويل أساسي، و LuaLaTeX لتنضيد PDF، ونصوص Python لتوليد الأنماط
ومعالجة ما بعد الإنتاج.

## الأهداف

- إنتاج مخرجات PDF و DOCX عالية الجودة من مصدر Markdown واحد.
- دعم المستندات ثنائية اللغة (العربية / الإنجليزية) مع معالجة صحيحة للنص
  ثنائي الاتجاه.
- توفير أنماط نشر مخصصة لكل دار نشر (افتراضي، O'Reilly، Apress).
- تخزين القطع الوسيطة مؤقتًا (رسوم Mermaid، صور الأغلفة) لبناء سريع
  متزايد.
- بنية معيارية نظيفة سهلة التوسع بإضافة صيغ إخراج أو أنماط جديدة.

## القدرات

| الميزة | الوصف |
|--------|-------|
| توليد PDF | PDF احترافي عبر Pandoc + LuaLaTeX مع سمات مخصصة |
| توليد DOCX | DOCX منسق عبر Pandoc مع قوالب مرجعية |
| رسوم Mermaid | عرض تلقائي لرسوم Mermaid إلى PNG |
| صفحات الأغلفة | أغلفة أمامية/خلفية من قوالب HTML باستخدام ImageMagick |
| معالجة أكواد C# | معالجة محافظة على تركيب أكواد C# |
| النص ثنائي الاتجاه | دعم كامل للاتجاه من اليمين لليسار ومن اليسار لليمين |
| أنماط النشر | ثلاثة أنماط مدمجة: افتراضي، O'Reilly، Apress |
| بناء متزايد | رسوم وأغلفة مخبأة تتجاوز إعادة التوليد |
| مساحة عمل مؤقتة | دليل مؤقت منعزل لكل تصدير، يُنظف تلقائيًا |

## لماذا هذا النظام؟

تم تقييم أدوات موجودة مثل PrinceXML و Typst ومحولات الويب لكنها افتقرت
إلى مجموعة الميزات المطلوبة:

- **Pandoc** يوفر أكثر تحويل ناضج من Markdown إلى كل الصيغ مع قابلية
  التوسع عبر مرشحات Lua.
- **LuaLaTeX** يقدم تنضيد PDF احترافي مع دعم `fontspec` للخطوط OpenType
  عبر النصوص العربية واللاتينية.
- **نصوص Python** تعالج معالجة ما بعد الإنتاج لـ DOCX (الهوامش، الأغلفة،
  تطبيع ألوان التظليل) التي لا يستطيع Pandoc وحده القيام بها.
- النظام المعياري القائم على PowerShell مصمم للصيانة والتوسع دون الحاجة
  إلى معرفة عميقة بكل مكون.
